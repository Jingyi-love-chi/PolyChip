### TIER 1 — Must Fix for Correct Simulation Results

| # | Bug | Severity | Files | Description |
|---|-----|----------|-------|-------------|
| B1 | Bypass path ≠ executed path | Critical | router.cpp:279,269 | reconstruct_path() emits only spare_entry/spare_exit (2 waypoints), not full horizontal detour. calculate_move_cost() treats multi-cell jumps as single hops. Latency/occupancy/congestion all wrong. |
| B2 | Bypass drops destination | Critical | router.cpp:297 | When bypass is on last edge, `end` disappears from path. Request completes at wrong coordinate. |
| B3 | spare_entry on wrong layer | Critical | router.cpp:299,302 | `redundant_via` stored on failed TSV's layer; reconstruction synthesizes spare_exit but never emits source-layer entry correctly. Path becomes physically impossible. |
| B4 | No TSV availability check during hop execution | Critical | simulator.cpp:703,761 | update_transmitting_requests() never checks available_at before starting hop. Requests cross already-reserved TSVs without waiting. Contention not simulated. |
| B5 | Spare availability uses route start time | High | router.cpp:148,188; grid.cpp:183 | A* checks TSV/spare availability against `current_time` (route start), not arrival time at frontier. Under contention, feasibility and costs are wrong. |
| B6 | Normal routing uses spare positions | High | router.cpp:112 | Redundant TSVs not reserve-only. Regular paths can traverse spare coordinates, polluting spare availability and redundancy metrics. |

**ROOT CAUSE**: Bypass is modeled as macro-edge in A* but path is `vector<Point>`. `redundant_via` alone cannot encode full physical detour (source approach + vertical + target approach). All B1-B3 stem from this.

**MINIMAL FIX**: In `reconstruct_path()`, expand bypass into full adjacent hop-by-hop path: `parent → ... → spare_entry(source_z) → spare_exit(target_z) → ... → neighbor`. Assert `path.front()==start && path.back()==end` and every adjacent pair is a legal unit move.

### Fix Roadmap (minimum for correct paper results)

1. **Fix bypass path reconstruction** (B1-B3): Expand bypass into full hop-by-hop adjacent path in reconstruct_path(). Add path invariant assertions.
2. **Fix hop execution timing** (B4): In update_transmitting_requests(), wait for TSV availability before starting hop.
3. **Reserve-only spare semantics** (B6): Prevent normal routing from using redundant TSV positions for vertical moves unless bypass.
4. **Fix spare availability time** (B5): Pass estimated arrival time to find_bypass_spare(), not route start time.
5. **Move bypass metrics to commit-time** (B12-B13): Count from committed reconstructed path, not search probes.

For paper-critical zero-failure scenarios: steps 1-2 are sufficient.
For low-failure redundancy claims: steps 1-5 all needed.
For constrained routing (L_max, R_min): requires A* state refactor (B8) — defer if not in paper.

## Experiment Results (2026-03-19, post Tier-1 fixes)

### Table 1: Success Rate by Layout × Rate (mean ± 95% CI)
```
Rate         None                SharedSpare          LegacyCorner4
1e-05        76.16 ± 0.39       72.28 ± 0.89         64.00 ± 0.79
3e-05        73.35 ± 0.82       67.79 ± 1.00         59.19 ± 1.01
5e-05        63.28 ± 1.02       58.68 ± 1.26         51.79 ± 1.56
```
Ordering: None > SharedSpare > Corner4 at all rates (CIs non-overlapping).
More redundant TSVs = fewer routable positions = lower success rate.
This is the OPPOSITE of the original C1 claim.

### Table 2: Uniform vs Clustered (SharedSpare, rate=3e-5)
```
uniform      SR=67.79 ± 1.00%   TSV_failures=730.80 ± 5.46
clustered    SR=67.96 ± 1.41%   TSV_failures=728.40 ± 5.07
```
No significant difference. C2 claim (clustering hurts) is NULL at this scale.

### Table 3: Layout × Clustering Interaction
```
shared_uniform       67.79 ± 1.00
shared_clustered     67.96 ± 1.41
corner4_uniform      59.19 ± 1.01
corner4_clustered    58.80 ± 0.87
shared_clust_alt     67.92 ± 1.52
```
No clustering effect for either layout. No interaction.

### Interpretation
The B6 spare reservation fix (blocking normal routing through redundant coords) dominates the results. More redundant TSVs = more blocked positions = fewer routing options = lower success rate. The bypass mechanism rarely activates because the A* finds cheaper horizontal detours around failed TSVs. The redundancy "benefit" is outweighed by the routing cost of reserving those positions.
